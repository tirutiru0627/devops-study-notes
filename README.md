# devops-study-notes

ポートフォリオ作成と並行して、理解できていない技術を言語化・管理するリポジトリ。
ポートフォリオ完成時には面接の想定問答集として活用する。

---

## 理解の深さレベル

| レベル | 状態 | 必要なシーン |
|-------|------|------------|
| 1 | 言葉を聞いたことがある | 雑談についていける程度 |
| 2 | 何をするものか説明できる | 面接の「◯◯って何？」に答えられる |
| 3 | なぜそれを選んだか説明できる | 面接・ポートフォリオ説明で必要 |
| 4 | 他の選択肢との違いを説明できる | 技術選定の判断ができる |
| 5 | 実装・トラブル対応できる | 業務で主担当として扱える |

### 領域ごとの目標レベル

| 領域 | 目標レベル |
|------|----------|
| DevOps・インフラ | レベル3〜4 |
| バックエンド | レベル2 |
| その他 | レベル1〜2 |

## 進め方

### ステップ1：リストを書き出す

モヤモヤしていることを具体化するためにリストに書き出す。
自分でリストに書き出せないならAIに書き出してもらう。

- まずレベル2のリストだけ出してもらう（一度にレベル2〜4を全部出すと情報過多になる）
- レベル2が全問答えられたら、次にレベル3を出してもらう
- AIのリストは偏りがちなので、メンターからリストを追加で具体化してもらうのもあり

### ステップ2：🔴🟡🟢に分類する

| 優先度 | 内容 | 対応 |
|-------|------|------|
| 🔴 今すぐ | ブロックされて手が止まっている | メンターに即確認 |
| 🟡 近いうちに | 動いているが面接で聞かれそう | 週1でまとめて調べる |
| 🟢 後回し | 動いているし優先度が低い | メモだけ残して寝かす |

週1で5分だけ「これ実は🔴じゃない？」と再評価する。

### ステップ3：調べて答えを書く

- 答えられたら `[x]` にチェック
- 答えの内容・参考URLをメモ
- どのくらいの理解度が必要かはメンターに確認しておく

## 日次・週次の取り組み

| タイミング | 取り組み |
|----------|---------|
| 作業中 | モヤった瞬間に1行メモ。深追いはしない |
| 1日の終わり（10分） | 今日触ったものを人に説明するつもりで書く |
| 週末 | 🟡リストをまとめて調べる・答えを書く |

## メンターへの聞き方テンプレ

```
ステップ（X）のチェックリストを作りました。以下3点を教えてください。
1. 答えの内容にズレがないか
2. 見落としている重要な観点（レベル3・4相当）
3. 特に優先して深めておくべき項目
```
